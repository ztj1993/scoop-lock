# EasyTier GUI 使用指南

## 简介

EasyTier 是一个简单、安全、去中心化的 Mesh VPN 网络解决方案。GUI 版本提供了图形用户界面，使配置和管理更加便捷。

**版本**: 2.3.2  
**官网**: https://github.com/EasyTier/EasyTier  
**许可证**: Apache-2.0

## 主要功能

- 去中心化 Mesh VPN
- 点对点 (P2P) 连接
- 自动 NAT 穿透
- 加密通信
- 跨平台支持
- 图形用户界面
- 简单配置

## 安装和启动

### 安装

```bash
# 通过 Scoop 安装
scoop install easytier-gui

# 或手动下载安装
# 从 GitHub Releases 下载最新版本
```

### 启动

```bash
# 启动 GUI 版本
easytier-gui.exe
```

## 网络配置

### 创建网络

1. 点击"创建网络"
2. 输入网络名称
3. 设置网络密码
4. 选择网络类型
5. 点击"确定"

### 加入网络

1. 点击"加入网络"
2. 输入网络名称
3. 输入网络密码
4. 输入网络地址（可选）
5. 点击"确定"

### 网络设置

```
网络名称: 网络的唯一标识
网络密码: 用于身份验证
网络地址: 节点的虚拟 IP 地址
监听端口: 监听连接的端口
```

## 节点管理

### 查看节点

```
在主界面查看已连接的节点
显示节点名称、IP 地址、连接状态
```

### 节点操作

```
连接节点: 双击节点
断开节点: 右键 → 断开
查看详情: 右键 → 查看详情
```

### 节点状态

```
在线: 节点已连接
离线: 节点未连接
连接中: 正在建立连接
```

## 网络拓扑

### 查看拓扑

```
1. 点击"网络拓扑"标签
2. 查看节点之间的连接关系
3. 查看连接质量
```

### 拓扑类型

```
全连接: 所有节点直接连接
星型: 通过中心节点连接
网状: 部分节点直接连接
```

## 安全设置

### 加密设置

```
启用加密: 默认启用
加密算法: ChaCha20-Poly1305
密钥交换: X25519
```

### 防火墙设置

```
允许入站连接: 是
允许出站连接: 是
监听端口: 默认 11010
```

## 高级配置

### 配置文件

```json
{
  "network_name": "my-network",
  "network_secret": "my-password",
  "ip_address": "10.0.0.1",
  "listeners": [
    "tcp://0.0.0.0:11010",
    "udp://0.0.0.0:11010"
  ],
  "peers": [
    "tcp://peer1.example.com:11010",
    "tcp://peer2.example.com:11010"
  ],
  "console_log_level": "info",
  "file_log_level": "info"
}
```

### 命令行参数

```bash
# 指定配置文件
easytier-gui.exe -c config.json

# 指定日志级别
easytier-gui.exe --log-level debug

# 指定监听端口
easytier-gui.exe --port 11010
```

## 常见问题

### 1. 无法连接到网络

```
1. 检查网络名称和密码是否正确
2. 检查防火墙设置
3. 检查网络连接
4. 查看日志获取详细信息
```

### 2. 连接速度慢

```
1. 检查网络带宽
2. 尝试不同的节点
3. 检查 NAT 类型
4. 启用中继模式
```

### 3. 节点无法发现

```
1. 确保网络名称和密码一致
2. 检查监听端口
3. 检查防火墙设置
4. 尝试手动添加节点
```

### 4. 如何提高安全性？

```
1. 使用强密码
2. 启用加密
3. 定期更换密码
4. 限制网络访问
```

## 故障排除

### 诊断工具

```bash
# 查看网络状态
easytier-cli status

# 查看节点信息
easytier-cli peers

# 测试连接
easytier-cli ping <节点IP>

# 查看日志
easytier-cli logs
```

### 常见错误

```
错误: 无法连接到对等节点
解决: 检查网络连接和防火墙设置

错误: 网络名称已存在
解决: 使用不同的网络名称

错误: 认证失败
解决: 检查网络密码
```

## 集成和扩展

### API 接口

```bash
# 获取网络状态
curl http://localhost:11010/api/status

# 获取节点列表
curl http://localhost:11010/api/peers

# 添加节点
curl -X POST http://localhost:11010/api/peers -d '{"address": "tcp://peer:11010"}'
```

### 脚本自动化

```bash
#!/bin/bash
# 自动连接脚本

# 启动 EasyTier
easytier-gui.exe -c config.json

# 等待连接
sleep 10

# 检查连接状态
curl http://localhost:11010/api/status
```
