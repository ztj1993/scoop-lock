# Navicat Premium 16 使用指南

## 简介

Navicat Premium 是一款功能强大的多数据库管理和开发工具，支持同时连接到多种不同类型的数据库。它提供了直观的图形界面，使数据库管理员和开发者能够高效地进行数据库管理、开发和维护工作。

- **版本**: Navicat Premium 16
- **许可证**: 商业软件（需购买许可证）
- **官方网站**: https://www.navicat.com.cn/products/navicat-premium

### 支持的数据库

Navicat Premium 16 支持以下数据库系统：

| 数据库 | 说明 |
|--------|------|
| MySQL | 世界上最流行的开源关系数据库 |
| MariaDB | MySQL 的分支，完全兼容 |
| PostgreSQL | 功能强大的开源对象关系数据库 |
| MongoDB | 流行的 NoSQL 文档数据库 |
| Oracle | 企业级关系数据库管理系统 |
| SQL Server | 微软的关系数据库管理系统 |
| SQLite | 轻量级嵌入式关系数据库 |
| Redis | 高性能键值存储数据库（Premium 16 新增） |

## 主要功能

### 数据库连接管理

Navicat Premium 提供了统一的连接管理界面：

- 支持同时连接多种类型的数据库
- 连接分组管理
- 支持 SSH 隧道和 HTTP 隧道连接
- SSL/TLS 安全连接
- 连接配置导入导出
- 连接颜色标记

### 数据查看与编辑

- **网格视图**: 以表格形式查看和编辑数据
- **表单视图**: 以表单形式查看单条记录
- **文本视图**: 查看大型文本/BLOB 数据
- **内联编辑**: 直接在网格中编辑数据
- **批量编辑**: 同时修改多条记录
- **数据筛选**: 按条件筛选显示数据
- **数据排序**: 按列排序数据

### SQL 编辑器

强大的 SQL 编辑功能：

- **语法高亮**: 支持多种数据库的 SQL 语法高亮
- **代码补全**: 智能 SQL 代码自动补全
- **代码片段**: 预定义和自定义 SQL 代码片段
- **SQL 格式化**: 自动格式化 SQL 语句
- **执行计划**: 查看 SQL 查询的执行计划
- **多标签编辑**: 同时打开多个 SQL 编辑窗口

### 数据库设计

可视化的数据库设计工具：

- **ER 图表**: 生成和编辑实体关系图
- **表设计器**: 可视化创建和修改表结构
- **外键关系**: 图形化管理表之间的关系
- **逆向工程**: 从现有数据库生成模型
- **正向工程**: 从模型生成数据库脚本

### 数据传输与同步

- **数据传输**: 在不同数据库之间传输数据
- **结构同步**: 比较和同步数据库结构
- **数据同步**: 比较和同步表数据
- **导出向导**: 将数据导出为多种格式
- **导入向导**: 从多种格式导入数据

### 数据导入导出

支持多种格式：

- **导入**: CSV、Excel、JSON、XML、SQL、Parquet、Access 等
- **导出**: CSV、Excel、JSON、XML、HTML、PDF、SQL 等
- **批量导入导出**: 支持多个表同时操作

- **表优化**: 优化表结构和性能
- **索引管理**: 创建、修改和删除索引

- **多种图表类型**: 柱状图、折线图、饼图、散点图等

## 常用命令/操作

### 安装

通过 scoop 安装：

```bash
scoop install navicat16
```

首次运行需要激活许可证。

### 创建数据库连接

1. 点击工具栏的 "连接" 按钮或菜单 "文件" -> "新建连接"
2. 选择数据库类型（如 MySQL、PostgreSQL 等）
3. 填写连接信息：
   - **连接名**: 自定义名称
   - **主机**: 数据库服务器地址
   - **端口**: 数据库端口（如 MySQL 默认 3306）
   - **用户名**: 数据库用户名
   - **密码**: 数据库密码
   - **默认数据库**: 可选，指定默认打开的数据库
4. 点击 "测试连接" 验证配置
5. 点击 "确定" 保存连接

### 浏览数据库

1. 在左侧导航面板中展开连接
2. 展开数据库节点查看对象类型：
   - 表（Tables）
   - 视图（Views）
   - 存储过程（Stored Procedures）
   - 函数（Functions）
   - 触发器（Triggers）
   - 事件（Events）
3. 双击表名打开数据查看器

### 执行 SQL 查询

1. 右键点击数据库，选择 "新建查询" 或按 Ctrl+Q
2. 在 SQL 编辑器中输入 SQL 语句
3. 使用 Ctrl+Shift+R 获取代码补全提示
4. 点击 "执行" 按钮或按 Ctrl+R 执行查询
5. 在结果面板中查看查询结果

### 创建新表

1. 右键点击 "表" 节点，选择 "新建表"
2. 在表设计器中：
   - 添加列：设置列名、数据类型、长度、默认值等
   - 设置主键：选择列并标记为主键
   - 添加索引：在索引标签页中配置
   - 设置外键：在外键标签页中配置关系
3. 点击 "保存" 创建表

### 编辑表数据

1. 双击表名打开数据网格
2. 点击单元格直接编辑
3. 右键行号选择操作：
   - 添加新记录
   - 复制记录
   - 删除记录
4. 编辑完成后点击 "保存" 按钮（Ctrl+S）

### 数据导入

1. 右键点击数据库或表，选择 "导入向导"
2. 选择导入源文件格式（如 CSV、Excel）
3. 选择源文件
4. 配置导入选项：
   - 分隔符设置
   - 字段映射
   - 数据转换规则
5. 预览导入数据
6. 执行导入

### 数据导出

1. 选择要导出的表或查询结果
2. 右键选择 "导出向导"
3. 选择导出格式（如 CSV、Excel、JSON）
4. 配置导出选项
5. 选择导出目标路径
6. 执行导出

### 数据库备份

1. 右键点击数据库，选择 "转储 SQL 文件" -> "结构和数据"
2. 选择保存路径
3. 配置备份选项（如是否包含 DROP 语句）
4. 执行备份

### 数据库还原

1. 右键点击数据库，选择 "运行 SQL 文件"
2. 选择备份文件
3. 配置执行选项
4. 执行还原

### 数据传输

1. 菜单选择 "工具" -> "数据传输"
2. 选择源连接和数据库
3. 选择目标连接和数据库
4. 选择要传输的表
5. 配置传输选项
6. 执行传输

### 结构同步

1. 菜单选择 "工具" -> "结构同步"
2. 选择源数据库和目标数据库
3. 点击 "比较" 查看差异
4. 选择要同步的对象
5. 预览同步脚本
6. 执行同步

## 常见问题

### Q: 连接数据库时报错 "Access denied" 怎么办？

- 确认用户名和密码正确
- 确认数据库用户有从当前主机连接的权限
- 检查 MySQL 的 `bind-address` 配置
- 确认防火墙允许数据库端口

对于 MySQL，可能需要授权：

```sql
GRANT ALL PRIVILEGES ON *.* TO 'username'@'%' IDENTIFIED BY 'password';
FLUSH PRIVILEGES;
```

### Q: 如何解决中文乱码问题？

- 确保数据库字符集设置正确（如 utf8mb4）
- 在连接属性中设置正确的字符集
- 在 "高级" 选项卡中设置编码为 UTF-8
- 检查表和列的字符集设置

### Q: 查询结果数据量很大时很卡怎么办？

- 使用 LIMIT 限制返回行数
- 在 "选项" 中调整每次加载的行数
- 使用筛选条件减少数据量
- 考虑为查询字段添加索引

### Q: 如何在不同数据库之间同步数据？

使用 Navicat 的 "数据传输" 功能：

1. 打开 "工具" -> "数据传输"
2. 选择源数据库（支持不同类型）
3. 选择目标数据库
4. 配置映射规则
5. 执行传输

### Q: 如何调试存储过程？

1. 在导航面板中找到存储过程
2. 右键选择 "调试"（部分数据库支持）
3. 设置断点
4. 输入参数值
5. 逐步执行并查看变量值

注意：调试功能仅支持部分数据库类型。

### Q: 如何备份 Navicat 的连接配置？

1. 菜单选择 "文件" -> "导出连接"
2. 选择要导出的连接
3. 保存为 .ncx 文件

恢复配置：

1. 菜单选择 "文件" -> "导入连接"
2. 选择 .ncx 文件
3. 导入完成

### Q: 如何使用 SSH 隧道连接远程数据库？

在连接属性中：

1. 切换到 "SSH" 标签页
2. 勾选 "使用 SSH 通道"
3. 填写 SSH 主机、端口（默认 22）
4. 填写 SSH 用户名和密码（或使用密钥）
5. 测试 SSH 连接
6. 确定保存
