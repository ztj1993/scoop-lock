# SourceGit 使用指南

## 简介

SourceGit 是一款开源的 Git 图形化客户端，为 Git 用户提供直观易用的界面来管理代码仓库。它支持 64 位和 ARM64 架构，是替代命令行 Git 操作的优秀选择。

**版本**: v2026.11  
**官网**: https://github.com/sourcegit-scm/sourcegit  
**许可证**: MIT  
**可执行文件**: SourceGit.exe  
**支持架构**: x64, ARM64

## 主要功能

- 直观的图形化界面，简化 Git 操作
- 支持所有标准 Git 操作（提交、推送、拉取、合并等）
- 可视化的分支管理和合并工具
- 内置差异比较工具，支持文件对比
- 支持多个仓库同时管理
- 支持 Git 子模块
- 支持 Git LFS (大文件存储)
- 支持交互式变基 (Interactive Rebase)
- 支持储藏 (Stash) 操作
- 支持标签管理
- 支持远程仓库管理
- 支持 Git 钩子 (Hooks) 配置

## 常用操作

### 启动 SourceGit

```bash
# 直接运行
SourceGit.exe

# 打开指定仓库
SourceGit.exe -r path\to\repository
```

### 克隆仓库

1. **点击"克隆仓库"按钮**
2. **输入远程仓库 URL**: 
   - HTTPS: `https://github.com/user/repo.git`
   - SSH: `git@github.com:user/repo.git`
3. **选择本地目录**: 选择仓库保存位置
4. **点击"克隆"**: 等待克隆完成

### 创建新仓库

1. **点击"新建仓库"按钮**
2. **选择目录**: 选择或创建新目录
3. **初始化仓库**: 点击"创建"按钮

### 基本提交流程

1. **查看更改**: 在"工作树"视图中查看未暂存的更改
2. **暂存文件**: 
   - 单个文件: 右键文件 → "暂存"
   - 所有文件: 点击"全部暂存"按钮
3. **输入提交信息**: 在提交信息框中输入描述
4. **提交**: 点击"提交"按钮
5. **推送到远程**: 点击"推送"按钮

### 分支管理

#### 创建分支

1. **点击"分支"菜单**
2. **选择"新建分支"**
3. **输入分支名称**
4. **选择起点**: 当前 HEAD 或指定提交
5. **点击"创建"**

#### 切换分支

1. **在分支列表中选择目标分支**
2. **双击或右键选择"检出"**

#### 合并分支

1. **切换到目标分支** (如 main)
2. **点击"合并"按钮**
3. **选择要合并的源分支**
4. **确认合并**

#### 删除分支

1. **右键点击要删除的分支**
2. **选择"删除分支"**
3. **确认删除**

### 远程操作

#### 添加远程仓库

1. **点击"远程"菜单**
2. **选择"管理远程"**
3. **点击"添加"**
4. **输入远程名称和 URL**

#### 推送更改

1. **点击"推送"按钮**
2. **选择远程仓库和分支**
3. **勾选"设置上游"以跟踪远程分支**
4. **点击"推送"**

#### 拉取更改

1. **点击"拉取"按钮**
2. **选择拉取选项**:
   - 拉取并合并
   - 拉取并变基
3. **点击"确定"**

### 差异比较

#### 查看工作树差异

1. **在"工作树"视图中选择文件**
2. **查看右侧的差异视图**
3. **绿色**: 新增内容
4. **红色**: 删除内容
5. **蓝色**: 修改内容

#### 比较提交

1. **在历史记录中选择提交**
2. **查看提交的详细信息**
3. **点击文件查看具体差异**

#### 比较分支

1. **右键点击分支**
2. **选择"与当前分支比较"**
3. **查看分支间的差异**

### 储藏操作

#### 储藏更改

1. **点击"储藏"按钮**
2. **输入储藏描述** (可选)
3. **选择是否包含未跟踪文件**
4. **点击"储藏"**

#### 恢复储藏

1. **在"储藏"视图中选择储藏**
2. **点击"应用储藏"**
3. **选择是否删除储藏**

### 标签管理

#### 创建标签

1. **右键点击提交**
2. **选择"创建标签"**
3. **输入标签名称**
4. **选择标签类型**:
   - 轻量标签: 仅指向提交
   - 附注标签: 包含额外信息
5. **点击"创建"**

#### 删除标签

1. **右键点击标签**
2. **选择"删除标签"**
3. **确认删除**

## 高级功能

### 交互式变基

1. **右键点击提交**
2. **选择"交互式变基"**
3. **在列表中调整提交顺序**
4. **选择操作**:
   - 选取 (pick): 保留提交
   - 压缩 (squash): 合并到前一个提交
   - 编辑 (edit): 暂停以修改提交
   - 删除 (drop): 删除提交
5. **点击"开始变基"**

### 子模块管理

#### 添加子模块

1. **点击"子模块"菜单**
2. **选择"添加子模块"**
3. **输入子模块 URL**
4. **选择本地路径**
5. **点击"添加"**

#### 更新子模块

1. **点击"子模块"菜单**
2. **选择"更新子模块"**
3. **选择更新选项**

### Git LFS

#### 初始化 LFS

1. **点击"LFS"菜单**
2. **选择"初始化 LFS"**

#### 跟踪文件

1. **点击"LFS"菜单**
2. **选择"跟踪文件"**
3. **输入文件模式** (如 `*.psd`)

## 界面导航

### 主界面布局

- **左侧**: 仓库列表和分支树
- **中间**: 提交历史和文件列表
- **右侧**: 差异查看器
- **底部**: 输出日志和状态信息

### 快捷键

- `Ctrl+N`: 新建仓库
- `Ctrl+O`: 打开仓库
- `Ctrl+S`: 暂存所有文件
- `Ctrl+Enter`: 提交
- `Ctrl+P`: 推送
- `Ctrl+L`: 拉取
- `Ctrl+D`: 查看差异
- `Ctrl+B`: 创建分支
- `Ctrl+M`: 合并
- `Ctrl+Z`: 撤销操作

### 用户配置

1. **点击"文件" → "设置"**
2. **在"用户"选项卡中配置**:
   - 用户名
   - 邮箱
   - 签名密钥 (GPG)

### 外观配置

1. **在"外观"选项卡中配置**:
   - 主题 (浅色/深色)
   - 字体大小
   - 语言设置
   - 日期格式

### 网络配置

1. **在"网络"选项卡中配置**:
   - 代理设置
   - SSH 客户端路径
   - HTTP/HTTPS 代理

## 常见问题

### 1. 无法克隆仓库

**可能原因**:
- 网络连接问题
- 仓库 URL 错误
- 认证失败

**解决方案**:
- 检查网络连接
- 验证仓库 URL 是否正确
- 配置 Git 凭据管理器
- 检查 SSH 密钥配置

### 2. 推送被拒绝

**可能原因**:
- 远程有新的提交
- 权限不足
- 分支保护规则

**解决方案**:
- 先拉取远程更改
- 检查仓库权限
- 联系仓库管理员

### 3. 合并冲突

**可能原因**:
- 同一文件被修改
- 删除了对方修改的文件

**解决方案**:
- 打开冲突文件
- 手动解决冲突标记
- 暂存解决后的文件
- 完成合并提交

### 4. 认证失败

**可能原因**:
- 用户名/密码错误
- SSH 密钥未配置
- 双因素认证问题

**解决方案**:
- 使用 Git 凭据管理器
- 配置 SSH 密钥
- 使用个人访问令牌 (PAT)

### 5. 大文件推送失败

**可能原因**:
- 文件超过 GitHub/GitLab 限制
- 未使用 Git LFS

**解决方案**:
- 使用 Git LFS 管理大文件
- 压缩文件后推送
- 使用 .gitignore 排除大文件

## 故障排除

### 查看日志

1. **点击"查看" → "日志"**
2. **查看详细的操作日志**
3. **复制错误信息进行搜索**

### 重置仓库状态

```bash
# 在终端中执行
git reset --hard HEAD
git clean -fd
```

### 修复损坏的仓库

```bash
# 检查仓库完整性
git fsck

# 修复损坏的对象
git gc --aggressive
```
