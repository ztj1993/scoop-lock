# CC-Switch 使用指南

## 简介

CC-Switch 是一款桌面端 All-in-One 助手工具，专为 Claude Code、Codex 和 Gemini CLI 设计。它提供了一个统一的图形界面，帮助开发者更高效地管理和切换不同的 AI 编程助手。

**版本**: 3.15.0
**官网**: https://github.com/farion1231/cc-switch
**许可证**: MIT
**类型**: GUI 桌面应用程序
**快捷方式**: cc-switch.exe

## 主要功能

- 统一管理多个 AI 编程助手
- 支持 Claude Code 集成
- 支持 Codex 集成
- 支持 Gemini CLI 集成
- 快速切换不同 AI 服务
- 图形化配置界面
- 会话历史记录
- 快捷键支持
- 系统托盘运行
- 自动更新检查

## 安装与启动

### 通过 Scoop 安装

```bash
scoop install cc-switch
```

### 启动应用程序

```bash
# 直接启动
cc-switch.exe

# 或通过开始菜单快捷方式
```

### 首次启动配置

1. 启动 CC-Switch 应用程序
2. 在设置界面配置 API 密钥
3. 选择默认的 AI 助手
4. 保存配置并重启应用

## 常用操作

### 配置 API 密钥

```
1. 点击菜单 "工具" → "设置"
2. 选择 "API 配置" 标签
3. 输入 Claude API Key
4. 输入 Codex API Key
5. 输入 Gemini API Key
6. 点击 "保存"
```

### 切换 AI 助手

```
方法一: 点击左侧面板的 AI 助手名称
方法二: 使用快捷键 Ctrl+1/2/3 切换
方法三: 使用工具栏下拉菜单选择
```

### 创建新会话

```
1. 点击 "文件" → "新建会话"
2. 或使用快捷键 Ctrl+N
3. 选择要使用的 AI 助手
4. 开始对话
```

### 管理会话历史

```
1. 点击 "视图" → "会话历史"
2. 查看历史会话列表
3. 双击打开历史会话
4. 右键删除或导出会话
```

## 快捷键列表

### 通用快捷键

```
Ctrl+N          新建会话
Ctrl+O          打开会话
Ctrl+S          保存会话
Ctrl+W          关闭当前会话
Ctrl+Q          退出程序
Ctrl+,          打开设置
F1              打开帮助
F5              刷新当前视图
```

### AI 助手切换

```
Ctrl+1          切换到 Claude Code
Ctrl+2          切换到 Codex
Ctrl+3          切换到 Gemini CLI
Ctrl+Tab        在 AI 助手间循环切换
```

### 编辑操作

```
Ctrl+C          复制
Ctrl+V          粘贴
Ctrl+X          剪切
Ctrl+Z          撤销
Ctrl+Y          重做
Ctrl+F          查找
Ctrl+H          替换
Ctrl+A          全选
```

## 配置文件

### 配置文件位置

```
Windows: %APPDATA%\cc-switch\config.json
```

### 配置文件示例

```json
{
  "general": {
    "language": "zh-CN",
    "theme": "dark",
    "auto_update": true,
    "minimize_to_tray": true
  },
  "ai_services": {
    "claude": {
      "enabled": true,
      "api_key": "your-claude-api-key",
      "model": "claude-3-opus",
      "max_tokens": 4096
    },
    "codex": {
      "enabled": true,
      "api_key": "your-codex-api-key",
      "model": "code-davinci-002"
    },
    "gemini": {
      "enabled": true,
      "api_key": "your-gemini-api-key",
      "model": "gemini-pro"
    }
  },
  "shortcuts": {
    "switch_claude": "Ctrl+1",
    "switch_codex": "Ctrl+2",
    "switch_gemini": "Ctrl+3"
  },
  "session": {
    "auto_save": true,
    "auto_save_interval": 300,
    "max_history": 100
  }
}
```

## 高级功能

### 系统托盘

```
1. 最小化时自动隐藏到系统托盘
2. 双击托盘图标打开主窗口
3. 右键托盘图标显示快捷菜单
4. 托盘菜单支持快速切换 AI 助手
```

### 插件扩展

```
1. 点击 "工具" → "插件管理"
2. 浏览可用插件列表
3. 安装所需插件
4. 配置插件参数
```

### 导入导出

```
导出会话:
1. 选择要导出的会话
2. 点击 "文件" → "导出"
3. 选择保存位置和格式

导入会话:
1. 点击 "文件" → "导入"
2. 选择要导入的文件
3. 确认导入
```

### 批量操作

```
1. 按住 Ctrl 选择多个会话
2. 右键选择批量操作
3. 支持批量删除、导出、归档
```

## 常见问题

### 1. 应用无法启动

```
可能原因:
- 系统不满足最低要求
- 缺少运行时依赖
- 配置文件损坏

解决方法:
1. 检查系统要求（Windows 10+）
2. 安装 Visual C++ Redistributable
3. 删除配置文件重新启动
4. 以管理员权限运行
```

### 2. API 密钥无效

```
可能原因:
- API 密钥输入错误
- API 密钥已过期
- 网络连接问题

解决方法:
1. 重新检查 API 密钥
2. 在对应平台重新生成密钥
3. 检查网络连接
4. 配置代理设置
```

### 3. 切换 AI 助手无响应

```
可能原因:
- 目标 AI 服务不可用
- 网络超时
- 本地缓存问题

解决方法:
1. 检查网络连接
2. 验证 API 密钥有效性
3. 清除本地缓存
4. 重启应用程序
```

### 4. 会话历史丢失

```
可能原因:
- 自动保存未启用
- 存储路径权限问题
- 磁盘空间不足

解决方法:
1. 启用自动保存功能
2. 检查存储路径权限
3. 清理磁盘空间
4. 定期手动备份
```

### 5. 界面显示异常

```
可能原因:
- DPI 缩放设置
- 主题兼容性问题
- 显卡驱动问题

解决方法:
1. 调整系统 DPI 设置
2. 切换到默认主题
3. 更新显卡驱动
4. 重置界面布局
```

### 6. 快捷键冲突

```
可能原因:
- 与其他软件快捷键冲突
- 系统快捷键占用

解决方法:
1. 在设置中自定义快捷键
2. 关闭冲突软件的快捷键
3. 使用不同的快捷键组合
```

## 故障排除

### 日志文件位置

```
Windows: %APPDATA%\cc-switch\logs\
```

### 启用调试模式

```
1. 编辑配置文件
2. 设置 debug: true
3. 重启应用程序
4. 查看详细日志
```

### 重置应用程序

```
1. 关闭 CC-Switch
2. 删除配置文件夹: %APPDATA%\cc-switch\
3. 重新启动应用程序
4. 重新配置 API 密钥
```

## 最佳实践

### API 密钥安全

```
1. 不要分享 API 密钥
2. 定期轮换密钥
3. 使用环境变量存储敏感信息
4. 启用 API 密钥的访问限制
```

### 性能优化

```
1. 定期清理会话历史
2. 限制同时打开的会话数
3. 关闭不使用的 AI 助手
4. 使用本地缓存加速
```

### 工作流建议

```
1. 为不同任务创建专用会话
2. 使用有意义的会话名称
3. 定期归档重要会话
4. 利用快捷键提高效率
```
